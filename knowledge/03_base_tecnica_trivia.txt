==================================================
TRÍVIA — BASE TÉCNICA OFICIAL
==================================================

[CATEGORIA DO SISTEMA]

Tipo: Sistema de atendimento inteligente via WhatsApp
Arquitetura: Webhook + Processamento de IA + Integração API Oficial Meta
Canal principal: WhatsApp Business API (Cloud API)
Modelo: SaaS modular escalável

==================================================

[ARQUITETURA GERAL]

A TRÍVIA opera em 4 camadas principais:

1) Camada de Entrada (Webhook)
- Recebe mensagens do WhatsApp via API oficial da Meta.
- Captura texto, áudio, mídia e metadados.
- Identifica número do cliente.
- Registra timestamp da interação.

2) Camada de Processamento
- Interpreta intenção da mensagem.
- Classifica tipo de solicitação.
- Consulta base de conhecimento (knowledge/*.txt).
- Aplica persona e regras comportamentais.
- Decide próximo passo do fluxo.

3) Camada de Inteligência
- Processamento via modelo de linguagem.
- Resposta humanizada.
- Aplicação de contexto.
- Aplicação de regras de segurança.
- Validação de escopo temático.

4) Camada de Saída
- Envio de resposta via Graph API.
- Registro interno.
- Acionamento de lead quando aplicável.

==================================================

[MODELO OPERACIONAL]

A TRÍVIA trabalha com:

- Atendimento orientado a intenção.
- Coleta estruturada de dados.
- Encaminhamento estratégico.
- Organização conversacional.

Fluxo base:

Mensagem recebida
↓
Identificação de intenção
↓
Consulta base institucional + técnica
↓
Aplicação de persona
↓
Geração de resposta
↓
Validação de lead
↓
Encaminhamento (se necessário)

==================================================

Tipos de fluxo configuráveis:

Cada fluxo contém:

Esses dados podem ser utilizados para:

==================================================

[SEGURANÇA E RESTRIÇÕES]

A TRÍVIA não pode:

- Inventar contatos.
- Inventar e-mails.
- Inventar preços.
- Enviar dados técnicos internos.
- Expor token, servidor ou estrutura.
- Sair do escopo do negócio.

Se cliente solicitar assunto fora do nicho:
- Responder educadamente.
- Redirecionar para tema atendimento/tecnologia.

==================================================

Modelo preparado para:

- Configuração personalizada por empresa.
- Modularização de recursos.

Cada cliente pode ter:

- Serviços ativados conforme plano.
